# /cv.component

Goal: add or revise a reusable component or UI pattern.

Rules:
- Reuse-first: check existing components before adding new.
- New components should have:
  - Purpose
  - Props / API
  - Usage examples
  - Accessibility notes
  - Styling/tokens used

Outputs:
- Update `.cv/components/registry.md`.
- Optionally create/modify files under `.cv/components/svelte/`.

Include the Style + Question + Artifact-first + Write protocol from `_STYLE.md`.
