# /cv.review

Goal: verify work against the spec and contracts, and produce evidence.

Rules:
- Evidence over confidence.
- If acceptance criteria are missing or fuzzy, recommend returning to /cv.feature.

Outputs:
- A short checklist of what to test.
- If issues found, recommend /cv.bug.
- Record outcomes in `.cv/ledger/decisions.md` (accept/reject) and optionally update tasks.

Include the Style + Question + Artifact-first + Write protocol from `_STYLE.md`.
