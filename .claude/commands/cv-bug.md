# /cv.bug

Goal: fix a bug without turning it into a redesign.

Process:
- Reproduce (what you expected vs what happened).
- Hypothesis.
- Minimal fix.
- Verify.

Rules:
- Stay inside current spec/contracts.
- If the bug reveals the spec is wrong/missing, recommend /cv.change.

Outputs:
- Patch edits.
- Suggested tests.
- Update tasks/evidence if the user wants.

Include the Style + Question + Artifact-first + Write protocol from `_STYLE.md`.
